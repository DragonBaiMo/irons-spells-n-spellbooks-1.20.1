# Iron's Spells 'n Spellbooks - 技能参数使用指南

*最后更新：2025-01-31*

## 快速开始

### 基础语法
```bash
# 玩家轨道：无参数，与卷轴效果完全一致
/cast @s <技能ID> <等级>

# 管理员轨道：带参数，完全自定义
/cast @s <技能ID> <等级> {"参数名": "参数值", ...}
```

### 双轨道设计

#### 玩家轨道 - 默认行为
```bash
/cast @s firebolt 5      # 与使用火球术卷轴效果100%相同
```
- 使用完全独立的原有逻辑
- 与卷轴/法术书施法100%一致
- 受配置文件限制
- 消耗魔法值、触发冷却
- 代码逻辑与原版完全相同

#### 管理员轨道 - 参数化施法
```bash
/cast @s firebolt 5 {"damage": 100.0}  # 自定义伤害
```
- 仅在通过 `/cast` 指令 + JSON 参数时调用
- 完全自定义的参数化施法
- 默认不消耗魔法值、不触发冷却
- 可使用目标选择器、自定义伤害、范围等

## 目标选择器参考

### 基础选择器
| 选择器 | 说明 | 示例 |
|--------|------|------|
| @self / @s | 自己 | @self |
| @target / @t | 看着的目标 | @target |
| @nearest | 最近的实体 | @nearest |
| @random | 随机实体 | @random |
| @all | 所有实体 | @all |

### 带参数的选择器
| 选择器 | 说明 | 示例 |
|--------|------|------|
| @player[name] | 指定名称的玩家 | @player[Notch] |
| @type[type] | 指定类型的实体 | @type[zombie] |

### 筛选条件
筛选条件放在方括号内，多个条件用逗号分隔：

| 条件 | 说明 | 示例 |
|------|------|------|
| type=<实体类型> | 实体类型 | type=zombie |
| radius=<距离> | 搜索半径 | radius=10 |
| health=<范围> | 生命值范围 | health=10-20 |
| name=<名称> | 实体名称 | name=Zombie |

### 复合选择器
使用 `&` 连接多个选择器：
- `@self&@nearest[5]` - 自己和5格内最近的实体
- `@all[type=zombie,radius=10]&@player[Notch]` - 10格内所有僵尸和Notch玩家

## 参数类型参考

| 类型 | JSON 格式 | 示例 | 说明 |
|------|-----------|------|------|
| INT | 数字 | 10 | 整数 |
| FLOAT | 数字 | 10.5 | 浮点数 |
| DOUBLE | 数字 | 10.123456 | 双精度 |
| BOOLEAN | 布尔 | true / false | 真/假 |
| STRING | 字符串 | "hello" | 文本 |
| VEC3 | 数组 | [100, 64, 200] | 三维坐标 |
| ENTITY_SELECTOR | 选择器 | "@target" | 实体选择器 |
| DIMENSION | 字符串 | "minecraft:the_nether" | 维度 ID |

## 注意事项

1. **权限要求**：需要OP 2级权限才能使用参数化施法
2. **JSON格式**：参数必须是有效的JSON格式
3. **自动类型转换**：数字类型会自动转换（int ↔ float ↔ double）
4. **双轨道分离**：
   - 无参数 = 卷轴效果（消耗魔法、有冷却、受配置限制）
   - 有参数 = 自定义效果（默认无消耗、无冷却、不受配置限制）
5. **管理员特权**：在管理员轨道中，除非明确指定，否则：
   - consumeMana: false（不消耗魔法值）
   - triggerCooldown: false（不触发冷却）
   - 完全跳过配置文件限制
